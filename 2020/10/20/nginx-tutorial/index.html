Nginx基础使用

发表于 2020-10-20 分类于 Nginx Valine：

Nginx基本功能

  1. 反向代理
  2. 负载均衡
  3. 动静分离

  反向代理，是指将请求发送给反向代理服务器，由反向代理服务器去选择目标服务器，获取数据后返回反向代理服务器，再返回客户端。如此，反向代理服务器和目标服务器在逻辑上就表现为同一个服务器，但是暴露的是反向代理服务器的IP而不是目标服务器的IP，换言之，隐藏了目标服务器的IP。

  正向代理，比如在天朝想访问Google，正常是访问不了的，但是可以在客户端配置一个代理服务器，由代理服务器去访问。

注意，反向代理和正向代理都需要配置代理服务器，请求也是发送给代理服务器，不同之处在于正向代理是在客户端进行配置，使用方是有感知的，而反向代理在服务端配置，对使用方是透明的。

Nginx基本命令

  • nginx -s stop 直接退出
  • nginx -s quit 退出并保存现场
  • nginx -s reload 热加载配置文件
  • nginx -c filename 使用指定的配置文件启动

Nginx配置文件

  Nginx配置文件主要由三部分组成：

  1. 全局块
  2. events块
  3. http块：全局块、server块

  其中，全局块是从配置文件开始到events块之间的内容，主要是一些影响Nginx整体运行的一些配置指令，包括用户组、worker process数量、进程pid file和日志文件存放路径以及其它配置文件的引入等。worker_processes是Nginx处理并发任务的关键配置，值越大，可以处理的并发任务也就越多。当然，这会受到物理机和操作系统状态的限制。

  events块主要涉及Nginx与用户之间的网络连接，常见配置包括是否开启对多worker processes下的网络连接进行序列化、是否允许同时接收多个网络连接、选取哪种事件驱动模型来处理连接请求和每个worker process可以支持的最大连接数等。

  http块是代理、缓存和日志定义等绝大多数功能和第三方模块配置的地方，它包括http全局块和server块。

  • http全局块的配置指令包括文件引入、MIME-TYPE定义、日志定义、指定连接超时时间和单链接请求数上限等
  • 每个http块可以包含多个server块，每个server块相当于一个虚拟主机。同时每个server块又可以分为全局server块和多个location块
    • 全局server块最常见的配置是当前主机的监听配置、当前主机的名称（或IP信息）
    • location块的主要作用是基于Nginx接收到的请求字符串（例如some_server/some_url），对当前主机名称（或IP别名）

之外的字符串（例如前面的/some_url）进行匹配，从而对特定的请求进行处理。同时，地址定向、数据缓存和应答控制等功能，还有许多第三方模块的配置也在这里进行

location指令详解

指令作用

  匹配指定的请求uri（请求uri不包含查询参数，如http://localhost:8080/test?id=10，请求uri是/test）

语法形式

匹配模式及顺序

匹配模式：

  1. 普通字符串（literal string）匹配
  2. 正则表达式（regular expression）匹配通字符串。

其中 ~ 和 ~* 用于正则表达式匹配， 其他前缀和无任何前缀都用于普通字符串匹配。

匹配顺序：

  1. 先匹配普通字符串，将最精确的匹配暂时存储；
  2. 然后按照配置文件中的声明顺序进行正则表达式匹配，只要匹配到一条正则表达式，则停止匹配，取正则表达式为匹配结果；
  3. 如果所有正则表达式都匹配不上，则取1中存储的结果；
  4. 如果普通字符串和正则表达式都匹配不上，则报404 NOT FOUND。

注意：正则匹配会根据匹配顺序，找到第一个匹配的正则表达式后将停止搜索。普通字符串匹配则无视顺序，只会选择最精确的匹配。
